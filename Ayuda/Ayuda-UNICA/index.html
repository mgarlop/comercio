Sistema Ayuda APP GUsuarios

Archivo de Ayuda

 

Esta aplicación web permite la gestión de usuarios que puede ser integrada en cualquier otra aplicación para el control de acceso a zonas restringidas. 

Presenta la particularidad de que hay que gestionar las imágenes de los usuarios porque se registran en una carpeta del servidor, por lo que hay que controlar el tipo y tamaño de archivo.

Otra particularidad es que los usuarios al ser borrados, no desaparecen, se pasan a una tabla HISTORICO como historial de todos los usuarios que han sido registrados en la aplicación. 

1. Inicio

Esta primera pantalla de la aplicación está construida con un encabezado y un pie, comunes a todos las páginas de forma que su funcionamiento se extiende a toda la aplicación. Después tenemos un cuerpo específico para cada página. 

El encabezado está compuesto por:

  • el título de la APP, "ESQUEMA DATOS DE USUARIOS", 
  • y la barra de navegación con cuatro enlaces:
    • INICIO. Lleva a esta misma página.
    • USUARIOS. Cambia a la página de gestión de usuarios.
    • HISTÓRICO. Cambia a la página de gestión del histórico.
    • AYUDA. Enlaza con este sistema de ayuda.

El pie muestra el logotipo del ciclo de ASIR y el texto "ASIR Turaniana" como firma de quienes desarrollan la APP.

El cuerpo de la página se sitúa en la zona central y muestra el texto "GESTIÓN DE USUARIOS" junto con una imagen que representa dos personas.

2. Gestión de usuarios

Esta pantalla de la aplicación mantiene el encabezado y pie, comunes a todas las páginas. 

En el cuerpo de esta página aparece el título de la página "Gestión de Usuarios" y una tabla con todos los datos registrados en el servidor de base de datos, donde hay que destacar:

  • El BOTÓN DE ALTAS, de color naranja y situado en la barra amarilla de encabezados de columnas. Se pulsará para abrir el formulario de recogida de datos de nuevos usuarios.
  • En cada fila, junto a la imagen de usuario, aparecen tres botones con las operaciones a realizar:
    • VER. Muestra todos los datos del usuario.
    • BORRAR. Elimina el usuario de la tabla de USUARIOS y lo copia al HISTÓRICO.
    • MODIFICAR. Permite editar los datos del usuario en cuestión y cambiarlos.

Tras la tabla nos muestra un paginador que permite navegar por las páginas que genera la consulta, cuando aparecen más de ocho registros por página.

2.1. Alta de usuarios

Pantalla de formulario de altas.

Recoge todos los datos para registrar nuevos usuarios en la tabla de USUARIOS del sistema. 

Hay que advertir que la imagen se sube al servidor desde el equipo y requiere comprobaciones de tipo y tamaño de archivo (admite formato JPG y PNG de hasta 200 Kb).

Esta pantalla lleva a un visor que informa si se ha registrado correctamente el usuario o si por el contrario ha habido errores. 

2.2. Ver datos de usuario

Pantalla VER datos de usuarios.

Muestra una ficha con los datos del usuario seleccionado. 

2.3. Eliminar usuario

Eliminar un usuario. 

Al ser eliminado un usuario del sistema, lo que hacemos es pasarlo al HISTÓRICO de modo que puede ser recuperado en cualquier momento. Una vez que muestra los datos del usuario registrado tenemos dos opciones en sendos botones:

  • Botón para confirmar el borrado.
  • Botón para cancelación y regresar a la pantalla de GUsuarios.

La eliminación de registros de esta pantalla implica que pasa todos los datos a la tabla HISTÓRICO y el archivo de imagen debe cambiar de carpeta.

2.4. Modificar usuario

Modificar datos de un usuario. 

El único dato que no permitimos cambiar del usuario es el NICK, que coincide con el DNI del usuario, el resto de datos, los podemos cambiar en alguna de las tres zonas en las que dividimos la pantalla:

  • Cambiar contraseña. Basta con pulsar el botón y accederemos a la pantalla para introducir nueva contraseña y repetirla. 

  • Cambiar datos del usuario. Basta con cambiar cualquier dato y pulsar el botón ENVIAR y regresa a la pantalla de GUsuarios. Con el botón RESTAURAR reinicia con los datos que tenemos almacenados.
  • Cambiar la imagen. Permite seleccionar nueva imagen y cambiarla pulsando el botón CAMBIAR IMAGEN.

3. Gestión del Histórico

Esta pantalla de la aplicación también mantiene el encabezado y pie, comunes a todas las páginas. 

En el cuerpo de esta página aparece el título de la página "Gestión de Usuarios Borrados" y una tabla con todos los datos registrados en el servidor de base de datos en la tabla HISTORICO, donde hay que destacar:

  • En cada fila, junto a la imagen de usuario, aparecen tres botones con las operaciones a realizar:
    • VER. Muestra todos los datos del usuario.
    • BORRAR. Elimina definitivamente los datos del usuario de la tabla HISTÓRICO.
    • RESTAURAR. Permite pasar el usuario a la tabla USUARIOS, desapareciendo de la tabla HISTORICO. Hay que precisar que la foto cambia de la carpeta "histórico" a la carpeta "imágenes", y por tanto cambia la ruta de la misma en el registro de la base de datos.

Tras la tabla nos muestra un paginador que permite navegar por las páginas que genera la consulta, cuando aparecen más de ocho registros por página.

3.1. Ver datos usuario del histórico

Pantalla VER datos de usuarios del histórico.

Muestra una ficha con los datos del usuario seleccionado. 

3.2. Borrar usuario del histórico

Eliminar un usuario del HISTÓRICO. 

Al ser eliminado un usuario del registro de históricos, se pierden definitivamente todos los datos sin posibilidad de recuperación. Elimina también la imagen de la carpeta de imágenes del histórico y no queda rastro de ese usuario por lo que se ha diseñado con un color que avisa del peligro que supone. Una vez que muestra los datos del usuario registrado tenemos dos opciones en sendos botones:

  • Botón para confirmar el borrado.
  • Botón para cancelación y regresar a la pantalla de GHistorico.
